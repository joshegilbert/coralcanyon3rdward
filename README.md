# Coral Canyon 3rd Ward Deacons Quorum Dashboard

A lightweight CRM / management tool for the Coral Canyon 3rd Ward Deacons Quorum (11-13 year olds) and their adult leaders. Replaces scattered spreadsheets and group texts with a single, role-aware home for the calendar, Sunday lessons, leader coverage, and announcements.

## Stack

- [Next.js 16](https://nextjs.org/) (App Router, Turbopack)
- TypeScript, Tailwind CSS v4, [shadcn/ui](https://ui.shadcn.com/) (base-nova preset on top of [Base UI](https://base-ui.com/))
- [Supabase](https://supabase.com/) — Postgres + Auth + Row Level Security
- [Lucide](https://lucide.dev/) icons, [date-fns](https://date-fns.org/), [zod](https://zod.dev/)

## Concepts

- **Roles** (RBAC, used by RLS): `adult_leader`, `youth`, `general`.
- **Callings** (purely organizational, NOT permissions): admin-managed many-to-many labels like `President`, `First Counselor`, `Sports Captain`, etc. Only youth can hold callings.
- **Calendar** (`events`): adult leaders edit; everyone reads.
- **Sunday Lesson** (`sunday_programs` + `program_blocks`): adult leaders AND youth can edit. Has a chrome-free [`/sunday-program/[id]/present`](app/(app)/sunday-program/[id]/present/page.tsx) view designed for following along during the lesson.
- **Leader RSVP** (`leader_rsvps`): private to adult leaders; each leader manages their own row. The home dashboard surfaces a red warning when fewer than 2 leaders are confirmed for a Sunday.

## Setup

### 1. Install dependencies

```bash
npm install
```

### 2. Environment variables

Copy `.env.local.example` to `.env.local` and fill in:

```
NEXT_PUBLIC_SUPABASE_URL=https://lenzoafaphijtfjandhr.supabase.co
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=sb_publishable_...
SUPABASE_SERVICE_ROLE_KEY=sb_secret_...    # for seed only, server-only
SUPABASE_ACCESS_TOKEN=...                  # for supabase CLI link/push
```

Grab the publishable key from Supabase Dashboard → Settings → API. The service-role key lives on the same page; treat it like a password. The access token is a personal token from [supabase.com/dashboard/account/tokens](https://supabase.com/dashboard/account/tokens).

### 3. Link to the Supabase project and push migrations

```bash
npx supabase link --project-ref lenzoafaphijtfjandhr
npm run db:push        # applies supabase/migrations/* to the cloud DB
npm run db:types       # regenerates lib/types/database.ts from the live schema
```

### 4. Seed realistic dev data

```bash
npm run seed
```

This creates 3 advisors, 6 youth, 2 parents, 4 callings, 8 events, and a sample Sunday program. Idempotent — safe to re-run. One Sunday is intentionally under-staffed so you can see the red warning on the dashboard.

### 5. Run dev server

```bash
npm run dev
```

Sign in at [http://localhost:3000/login](http://localhost:3000/login) with:

| Email                              | Role         | Password         |
| ---------------------------------- | ------------ | ---------------- |
| `advisor1@coralcanyon.local`       | adult_leader | `CoralCanyon!23` |
| `youth1@coralcanyon.local`         | youth        | `CoralCanyon!23` |
| `parent1@coralcanyon.local`        | general      | `CoralCanyon!23` |

## Folder Structure

```
app/
  (auth)/                  # /login, /signup with auth-only layout
  (app)/                   # shell-wrapped routes (sidebar + user menu)
    page.tsx               # role-aware home dashboard
    calendar/              # placeholder
    sunday-program/[id]/present/   # chrome-free follow-along view
    leaders/rsvp/          # adult_leader-only placeholder
    people/                # adult_leader-only placeholder
  auth/
    callback/route.ts      # magic-link / signup callback
    signout/route.ts       # POST -> signOut + redirect
components/
  ui/                      # shadcn primitives (base-nova)
  layout/                  # AppShell, Sidebar, UserMenu
  dashboard/               # Adult Leader Home Dashboard widgets
  shared/                  # CallingBadge, PersonLabel, PagePlaceholder
lib/
  types/database.ts        # generated by `npm run db:types`
  types.ts                 # friendly aliases (Profile, Calling, etc.)
  queries/dashboard.ts     # server-side fetchers for the home page
  sunday.ts                # 1st/3rd vs 2nd/4th classification
  auth.ts                  # getCurrentUser / requireAdultLeader
utils/supabase/            # createClient (server/browser/middleware/admin)
supabase/migrations/       # SQL schema, RLS policies, new-user trigger
scripts/seed.ts            # idempotent dev seed via service-role admin API
proxy.ts                   # Next 16 "proxy" (session refresh + auth gate)
```

## Deploying to Vercel

1. `vercel link` to create/link a Vercel project named `coralcanyon3rdward`.
2. Add the three Supabase env vars to Vercel:
   ```bash
   vercel env add NEXT_PUBLIC_SUPABASE_URL
   vercel env add NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY
   vercel env add SUPABASE_SERVICE_ROLE_KEY
   ```
3. In Supabase Dashboard → Authentication → URL Configuration:
   - Set **Site URL** to `https://coralcanyon3rdward.vercel.app`
   - Add to **Redirect URLs**: `https://coralcanyon3rdward.vercel.app/auth/callback`, `https://*.vercel.app/auth/callback`, and `http://localhost:3000/auth/callback`
4. `vercel --prod` for the initial deploy.
